Warum nicht HTML/CSS per MCP manipulieren?

Bei der klassischen Figma-to-Angular Pipeline wird HTML/CSS direkt per MCP-Server modifiziert. Fuer 3D-Szenen und Games ist dieser Ansatz fundamental ungeeignet:

web Traditionell: HTML/CSS

  • - DOM-basierte Darstellung
  • - CSS fuer Styling (2D-fokussiert)
  • - JavaScript Event Loop
  • - Browser Rendering Engine
  • - 60 FPS max (vsync limited)

view_in_ar Neu: Shader Pipeline

  • + Entity-Component System
  • + glTF fuer 3D Assets (native)
  • + Vulkan Render Pipeline
  • + GPU-native Shader Execution
  • + Unlimitierte Framerate

build Authoring-Pipeline vs. Runtime-Output

Wichtige Unterscheidung: HTML/CSS ist Runtime-Output fuer den Browser. Der MCP-Server in der Shader-Pipeline dient hingegen der Erstellung dieser Daten.

Web: MCP -> HTML/CSS

MCP generiert HTML/CSS, Browser fuehrt aus

MCP -> HTML/CSS -> Browser
3D: MCP -> glTF/Shader

MCP generiert Scene-Data, Vulkan fuehrt aus

MCP -> glTF/ECS -> Vulkan

Kern: Die MCP-Pipeline ist ein Authoring-Tool - sie erstellt/modifiziert die Daten, die dann von der jeweiligen Runtime (Browser oder Vulkan) konsumiert werden.

G glTF statt CSS: Die neue Styling-Basis

Waehrend CSS fuer 2D-Webseiten Farben, Layouts und Animationen definiert, uebernimmt glTF (GL Transmission Format) diese Rolle fuer 3D-Szenen. glTF ist das "JPEG der 3D" - ein standardisiertes Format fuer 3D-Assets.

CSS definiert:

  • - Farben, Gradienten
  • - Box Model, Layouts
  • - Transforms (2D/3D begrenzt)
  • - Transitions, Animations
  • - Schatten, Filter

glTF definiert:

  • + Meshes, Geometrien
  • + Materials (PBR-basiert)
  • + Texturen, Normal Maps
  • + Skeletal Animations
  • + Scene Hierarchy

S Fragment und Vertex Shader

Die AI manipuliert nicht HTML/CSS, sondern Shader-Parameter. Shader sind Programme die auf der GPU laufen und jeden Pixel (Fragment) bzw. jeden Vertex berechnen.

Vertex Shader

Transformiert Positionen, deformiert Geometry, berechnet Normals

  • - Input: Vertex Position, Normal, UV
  • - Output: Transformed Position
  • - AI kann: Deformation, Animation

Fragment Shader

Berechnet Farbe jedes Pixels, Lighting, Effects

  • - Input: Interpolated Data
  • - Output: Final Color (RGBA)
  • - AI kann: Farben, Glow, Mood

change_history Vertex Shader mit AI-Kontrolle 

#version 450

// Vertex Inputs
layout(location = 0) in vec3 inPosition;
layout(location = 1) in vec3 inNormal;
layout(location = 2) in vec2 inUV;

// Outputs to Fragment Shader
layout(location = 0) out vec3 fragNormal;
layout(location = 1) out vec2 fragUV;
layout(location = 2) out vec3 fragWorldPos;

// Uniforms - Standard Matrices
layout(set = 0, binding = 0) uniform CameraUBO {
    mat4 view;
    mat4 projection;
};

layout(set = 1, binding = 0) uniform ObjectUBO {
    mat4 model;
};

// ===== AI-CONTROLLED PARAMETERS =====
layout(push_constant) uniform AIParams {
    float deformation_amount;   // AI: 0.0 - 1.0
    float wave_frequency;       // AI: Wellenbewegung
    float time;                 // System: Animation Zeit
    float scale_pulse;          // AI: Pulsieren
} ai;

void main() {
    // AI-gesteuerte Deformation
    vec3 pos = inPosition;

    // Wellen-Effekt (AI steuert Frequenz)
    float wave = sin(pos.x * ai.wave_frequency + ai.time) * ai.deformation_amount;
    pos.y += wave;

    // Pulsierender Scale (AI steuert Intensitaet)
    float pulse = 1.0 + sin(ai.time * 2.0) * ai.scale_pulse;
    pos *= pulse;

    // Standard Transform
    vec4 worldPos = model * vec4(pos, 1.0);
    gl_Position = projection * view * worldPos;

    // Pass to Fragment
    fragNormal = mat3(model) * inNormal;
    fragUV = inUV;
    fragWorldPos = worldPos.xyz;
}

palette Fragment Shader mit AI-Kontrolle 

#version 450

// Inputs from Vertex Shader
layout(location = 0) in vec3 fragNormal;
layout(location = 1) in vec2 fragUV;
layout(location = 2) in vec3 fragWorldPos;

// Output
layout(location = 0) out vec4 outColor;

// Textures
layout(set = 2, binding = 0) uniform sampler2D albedoMap;
layout(set = 2, binding = 1) uniform sampler2D normalMap;

// ===== AI-CONTROLLED PARAMETERS =====
layout(push_constant) uniform AIParams {
    vec4 tint_color;            // AI: Farbton der Szene
    float emission_strength;    // AI: Glow-Intensitaet
    float saturation;           // AI: Farbsaettigung
    float contrast;             // AI: Kontrast
    float mood_dark;            // AI: Stimmung (0=hell, 1=dunkel)
} ai;

// Lighting
layout(set = 0, binding = 1) uniform LightUBO {
    vec3 lightDir;
    vec3 lightColor;
    vec3 ambientColor;
};

vec3 adjustSaturation(vec3 color, float sat) {
    float gray = dot(color, vec3(0.299, 0.587, 0.114));
    return mix(vec3(gray), color, sat);
}

vec3 adjustContrast(vec3 color, float c) {
    return (color - 0.5) * c + 0.5;
}

void main() {
    // Sample base texture
    vec4 albedo = texture(albedoMap, fragUV);

    // Simple diffuse lighting
    vec3 N = normalize(fragNormal);
    float NdotL = max(dot(N, -lightDir), 0.0);
    vec3 diffuse = albedo.rgb * lightColor * NdotL;
    vec3 ambient = albedo.rgb * ambientColor;

    vec3 color = diffuse + ambient;

    // ===== AI EFFECTS =====

    // 1. Tint (AI-controlled color overlay)
    color = mix(color, color * ai.tint_color.rgb, ai.tint_color.a);

    // 2. Emission/Glow (AI can make things glow)
    vec3 emissive = albedo.rgb * ai.emission_strength;
    color += emissive;

    // 3. Saturation adjustment (AI mood)
    color = adjustSaturation(color, ai.saturation);

    // 4. Contrast (AI dramatic effect)
    color = adjustContrast(color, ai.contrast);

    // 5. Mood darkening (AI atmosphere)
    color = mix(color, color * vec3(0.3, 0.3, 0.4), ai.mood_dark);

    outColor = vec4(color, albedo.a);
}

Die Kommunikation zwischen MCP-Server und C++ Vulkan Engine laeuft ueber gRPC Bidirectional Streams. Dies ermoeglicht:

Echtzeit-Updates

Kontinuierlicher Datenstrom ohne Polling

Bidirektional

AI sendet Commands, Engine sendet State

Low Latency

HTTP/2, Binary Protobuf, Multiplexing

Protocol Buffer Definition:

syntax = "proto3";

package shader_pipeline;

// ===== AI -> Engine Commands =====

message ShaderCommand {
  oneof command {
    SetMood set_mood = 1;
    ModifyEntity modify_entity = 2;
    TriggerAnimation trigger_animation = 3;
    SpawnEntity spawn_entity = 4;
  }
}

message SetMood {
  enum Mood {
    CALM = 0;
    AGGRESSIVE = 1;
    MYSTERIOUS = 2;
    JOYFUL = 3;
  }
  Mood mood = 1;
  float intensity = 2;
  float transition_seconds = 3;
}

message ModifyEntity {
  uint64 entity_id = 1;
  ShaderParams params = 2;
}

message ShaderParams {
  repeated float tint_color = 1;  // RGBA
  float emission = 2;
  float deformation = 3;
  float saturation = 4;
  float contrast = 5;
}

// ===== Engine -> AI State Updates =====

message SceneState {
  repeated EntityState entities = 1;
  float current_time = 2;
  uint32 frame_number = 3;
}

message EntityState {
  uint64 id = 1;
  repeated float position = 2;  // xyz
  float health = 3;
  bool visible = 4;
  string current_animation = 5;
}

// ===== Service Definition =====

service ShaderPipeline {
  // Bidirectional Stream fuer Echtzeit-Kommunikation
  rpc StreamControl(stream ShaderCommand) returns (stream SceneState);

  // Einmalige Abfragen
  rpc QueryEntities(EntityQuery) returns (EntityList);
  rpc LoadAsset(AssetRequest) returns (AssetResponse);
}

Die C++ Engine empfaengt AI-Commands via gRPC und setzt sie in Vulkan-Befehle um. Push Constants und Uniform Buffers uebertragen die Shader-Parameter zur GPU.

Vulkan Push Constants fuer AI-Parameter:

// ai_shader_params.hpp

#pragma once
#include <glm/glm.hpp>
#include <vulkan/vulkan.h>

// Muss mit GLSL layout uebereinstimmen!
struct AIShaderParams {
    glm::vec4 tint_color{1.0f};
    float emission_strength = 0.0f;
    float saturation = 1.0f;
    float contrast = 1.0f;
    float mood_dark = 0.0f;

    // Vertex shader params
    float deformation_amount = 0.0f;
    float wave_frequency = 1.0f;
    float time = 0.0f;
    float scale_pulse = 0.0f;
};

static_assert(sizeof(AIShaderParams) <= 128,
    "Push constants limited to 128 bytes");

// vulkan_renderer.cpp

void VulkanRenderer::updateAIParams(
    VkCommandBuffer cmd,
    const AIShaderParams& params
) {
    vkCmdPushConstants(
        cmd,
        m_pipelineLayout,
        VK_SHADER_STAGE_VERTEX_BIT | VK_SHADER_STAGE_FRAGMENT_BIT,
        0,
        sizeof(AIShaderParams),
        ¶ms
    );
}

// Render Loop mit AI-Updates
void VulkanRenderer::renderFrame() {
    // 1. Process pending AI commands from gRPC
    while (auto cmd = m_grpcClient->pollCommand()) {
        applyAICommand(*cmd);
    }

    // 2. Update time-based animations
    m_aiParams.time = m_timer.elapsed();

    // 3. Begin render pass
    VkCommandBuffer cmd = beginFrame();

    // 4. For each entity with AI control
    auto view = m_world.view();
    for (auto [entity, ai, mesh, params] : view.each()) {
        if (!ai.active) continue;

        // 5. Push AI params to GPU
        AIShaderParams gpu_params = convertToGPU(params);
        updateAIParams(cmd, gpu_params);

        // 6. Draw
        drawMesh(cmd, mesh);
    }

    endFrame();

    // 7. Send scene state back to AI
    m_grpcClient->sendSceneState(captureSceneState());
}
