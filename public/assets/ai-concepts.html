AI Behavior Trees (Markdown)

Behavior Trees definieren die Verhaltensregeln der AI in Markdown-Dateien. Sie steuern Entscheidungsprozesse, Workflows und Constraints.

Hierarchie

folder CLAUDE.md - Global Rules
description .claude/behavior.md - Project Rules
code .claude/commands/*.md - Actions

Behavior Workflow

1 LESEN - Existierenden Code verstehen
2 VERSTEHEN - Architektur analysieren
3 PLANEN - Minimale Änderungen definieren
4 UMSETZEN - Clean Code implementieren
5 VERIFIZIEREN - Build prüfen
// CLAUDE.md - Behavior Definition
## 1. VERSTEHEN VOR SCHREIBEN

### Pflicht vor jeder Code-Änderung:
- [ ] **Existierenden Code lesen**
- [ ] **Architektur respektieren**
- [ ] **Abhängigkeiten prüfen**
- [ ] **Interfaces studieren**

### Verboten:
- Code generieren ohne die Umgebung zu kennen
- Annahmen über nicht gelesene Dateien treffen
- Duplikate erstellen statt existierende Lösungen zu nutzen

Slash Commands

Slash Commands kodieren Multi-Step-Prozeduren als ausführbare Markdown-Dateien. Sie werden zur Laufzeit expandiert und als AI-Instruktionen interpretiert.

/sync-repos

Synchronisiert alle Git-Submodules

/get-current-state

Generiert HTML-Statusseite

/clean-workspace

Bereinigt Build-Artefakte

// .claude/commands/sync-repos.md
---
description: Clone or update all repositories from GitHub
---

## Instructions
1. Read .gitmodules to get the list of submodules
2. For each submodule, check if the directory exists
3. If not exists: `git submodule add <url> <path>`
4. If exists: `git submodule update --remote <path>`
5. Report status for each repository
